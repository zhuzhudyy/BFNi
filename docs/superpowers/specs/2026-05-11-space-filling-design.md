# 空间填充实验设计

## 1. 问题陈述

### 1.1 当前采集策略的局限

当前 BO 使用 EI 采集函数，只在高产率区域集中采样。这导致：
- 代理模型在未探索区域预测不可靠
- 对任意新样品的产率预测缺乏全局精度
- ARD 长度尺度在低采样区域估计不准

### 1.2 目标

补充空间填充采样策略，使模型在**整个特征空间**（Pre_ + Process_）上具备可靠的预测能力，而非仅在高产率局部区域。

---

## 2. 设计方案：混合采集（EI + 空间填充）

### 2.1 核心思路

使用 LHS 生成候选池，然后通过**加权打分**机制选择采样点：

```
score(x) = α × rank_EI(x) + (1-α) × rank_dist(x)
```

α=0.5 时 EI 和 maximin distance 各占一半权重。加权打分能找到"既有探索价值、产率预期也不错"的折中点，优于配额划分（50% 纯 EI + 50% 纯 distance 会选出两个极端）。

### 2.2 完整流程

```
1. 对 Pre_ 特征做 K-Means 聚类（k=3~5，样本量不足时降级处理）
2. 将待测新样品分配到各 cluster
3. 对每个 cluster：
   a. 生成 N_candidates 个 LHS 候选点（含 Process_ 参数）
   b. 拒绝采样：剔除 Ar < 2×H₂ 的不可行点
   c. 计算每个候选点的 EI 值
   d. 在归一化空间计算每个候选点到 X_train 的 maximin distance
   e. 计算综合得分 score(x)，贪心选取 N_new_i 个点（见 3.3 节）
4. 输出推荐采样方案
```

### 2.3 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 候选池生成 | LHS（非纯随机） | 保证初始覆盖均匀性 |
| 约束处理 | 拒绝采样（非投影） | 投影在 Ar=2×H₂ 边界创造人为数据墙，破坏均匀性；约束空间占比 ~70%，拒绝采样效率足够 |
| 分配比例 | 按新样品的 cluster 分布 | 按训练集分配会重复覆盖已充分采样的区域 |
| 选择机制 | 加权打分（α=0.5） | 找到 EI 和 distance 的折中点，避免极端选择 |

## 3. 数学细节

### 3.1 混合准则评分

对候选点 x，定义综合得分：

```
score(x) = α × rank_EI(x) + (1-α) × rank_dist(x)
```

其中：
- `rank_EI(x)`：EI 值在候选池中的排名百分位（0~1，越大越好）
- `rank_dist(x)`：maximin distance 在候选池中的排名百分位（0~1，越大越好）
- `α = 0.5`：混合系数

选择 score 最高的点作为下一个采样点。

**Rank 机制的已知妥协**：Rank 会抹杀 EI 的绝对幅度差异（EI=15.0 和 EI=0.5 在 rank 视角下差异微小）。这是 Exploration vs Exploitation 的固有妥协。当前目标是全局模型精度，接受此妥协。

### 3.2 Maximin Distance

```
d_min(x) = min_{x' ∈ X_train} ||x - x'||
```

即候选点到所有已训练样本的最小距离。优先选择距离已有点最远的候选点，保证空间覆盖。

**特征归一化**：距离计算必须在归一化后的特征空间进行。工艺参数尺度差异极大（Temp ~1000-1400, Ar ~800, Time ~1-30），原始尺度下 Temp/Ar 会完全主导距离。所有 Pre_ 和 Process_ 特征通过 Min-Max 缩放映射到 [0, 1] 后再计算距离。

### 3.3 批次采样的贪心更新

当一个 cluster 需要选出 N_new_i > 1 个点时，不能一次性取 Top N（会导致选出的点彼此靠得很近）。必须使用贪心更新：

```
X_virtual = X_train.copy()  # 虚拟扩展集
selected = []
for _ in range(N_new_i):
    对所有候选点计算 d_min(x) = min_{x' ∈ X_virtual} ||x - x'||
    计算 score(x) = α × rank_EI(x) + (1-α) × rank_dist(x)
    选 score 最高的点 x*
    selected.append(x*)
    X_virtual = vstack([X_virtual, x*])  # 虚拟加入，更新距离基准
    从候选池中移除 x*
```

这样每选一个点，后续候选点的距离基准都会更新，保证选出的点之间也有良好的空间分散性。

### 3.4 拒绝采样

对 Process_ 空间 (Temp, Time, H₂, Ar)：
1. LHS 生成候选点
2. 剔除 Ar < 2×H₂ 的点
3. 保留可行点

约束空间占比估计：
- H₂ ∈ [0, H₂_max], Ar ∈ [0, Ar_max]
- 可行区域：Ar ≥ 2×H₂
- 若 H₂ 和 Ar 独立均匀分布，可行占比 ≈ 1 - 2×H₂_max/Ar_max
- 实际工艺范围下可行占比 > 70%，拒绝采样效率足够

## 4. 实现细节

### 4.1 Cluster 数量选择

使用 silhouette score 自动选择 k ∈ {3, 4, 5}。**小样本防御**：若训练样本量 < 15，跳过聚类，将整个 Pre_ 空间视为单一 cluster（k=1）。

```python
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

if len(X_pre_train) < 15:
    k_best = 1  # 样本不足，不聚类
else:
    scores = {}
    for k in [3, 4, 5]:
        km = KMeans(n_clusters=k, random_state=42).fit(X_pre_train)
        scores[k] = silhouette_score(X_pre_train, km.labels_)
    k_best = max(scores, key=scores.get)
```

### 4.2 候选池大小

每个 cluster 生成 1000 个 LHS 候选点（拒绝采样后保留 ~700 个）。

### 4.3 与 EI 的集成

在 `suggest_next()` 方法中：
1. 如果是空间填充轮次：使用上述混合准则
2. 如果是常规轮次：使用纯 EI
3. 空间填充轮次占比：前 N_fill 轮使用混合准则，之后切换到纯 EI

## 5. 与 Gemini 反馈的对齐

| Gemini 建议 | 采纳情况 | 说明 |
|-------------|---------|------|
| LHS 作为候选池 + EI 筛选 | 部分采纳 | 采纳候选池思路，但不纯用 EI，而是加权打分 |
| 拒绝采样替代投影 | 完全采纳 | 投影破坏均匀性，拒绝采样更优 |
| 按新样品分布分配 | 完全采纳 | 按训练集分配会重复覆盖已充分采样的区域 |
| 统一加权打分机制 | 完全采纳 | 2.2 节配额划分已修正为 3.1 节的加权打分 |
| 贪心更新避免距离短视 | 完全采纳 | 新增 3.4 节，选点后虚拟加入 X_train 再重算距离 |
| 聚类小样本防御 | 完全采纳 | 4.1 节加入 len < 15 时跳过聚类的防御逻辑 |

## 6. 风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|------|---------|
| 混合比例 α 不合适 | 中 | α 作为可调参数，默认 0.5，可根据实际效果调整 |
| 拒绝采样效率低 | 低 | 约束空间占比 > 70%，效率足够 |
| Cluster 数量不合适 | 低 | 使用 silhouette score 自动选择 |
| 空间填充轮次过多导致优化效率低 | 中 | 空间填充轮次有上限，之后切换纯 EI |
